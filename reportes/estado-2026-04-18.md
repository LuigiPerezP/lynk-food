## 1. Resumen Ejecutivo

LynkFood es una aplicación web de menú digital QR para restaurantes. El cliente escanea un código QR en su mesa, visualiza el menú completo con fotos y descripciones, arma su pedido con comentarios por ítem, lo envía, y puede hacer seguimiento en tiempo real del estado de cocina — sin necesidad de app nativa ni registro de cuenta.

El proyecto está en producción activa sobre Vercel, con base de datos y tiempo real en Supabase. Ha pasado de un MVP básico a un sistema con flujo de pedido completo end-to-end funcional. El desarrollo sigue en curso con funcionalidades de pulido pendientes.

**Estado general:** ✅ Producción estable · Funcionalidades core completas · Backlog de mejoras activo

## 4. Modelo de Datos (Supabase / PostgreSQL)

### Tablas activas

**`restaurantes`**
```sql
id          TEXT PRIMARY KEY
nombre      TEXT
logo        TEXT
activo      BOOLEAN
```

**`categorias`**
```sql
id              UUID PRIMARY KEY
restaurante_id  TEXT REFERENCES restaurantes
nombre          TEXT
orden           INTEGER
parent_id       UUID REFERENCES categorias  -- NULL = sección, UUID = subcategoría
```
Estructura jerárquica de 2 niveles: Sección → Subcategoría. Los ítems del menú se asignan a categorías hoja (subcategorías o secciones sin hijos).

**`menu_items`**
```sql
id              UUID PRIMARY KEY
restaurante_id  TEXT REFERENCES restaurantes
nombre          TEXT
descripcion     TEXT
precio          DECIMAL
categoria       TEXT  -- nombre de la categoría hoja
disponible      BOOLEAN
emoji           TEXT
imagen          TEXT  -- URL pública de Supabase Storage
```

**`pedidos`**
```sql
id              UUID PRIMARY KEY
restaurante_id  TEXT REFERENCES restaurantes
mesa            INTEGER
items           JSONB  -- [{menuItemId, nombre, cantidad, precio, emoji, nota?}]
notas           TEXT   -- nota general del pedido
estado          TEXT   -- 'nuevo' | 'preparando' | 'listo' | 'entregado'
created_at      TIMESTAMPTZ
updated_at      TIMESTAMPTZ
```

### Storage

**Bucket `menu-images`** (público):
- Archivos: JPG únicamente, sin límite de tamaño configurado
- Políticas RLS: `anon` puede INSERT, SELECT, UPDATE (necesario para carga desde cliente)
- URL pública: `https://cqdkkhlkjpdbpgwpaest.supabase.co/storage/v1/object/public/menu-images/...`

## 5. Flujo de Usuario Completo

### 5.1 Cliente (mesa)

1. **Escanea QR** → navega a `/mesa/[numero]`
2. **Explora menú**: filtra por sección (tabs) y subcategoría (chips horizontales)
3. **Agrega ítems**: tap `+`, opcionalmente agrega nota por ítem
4. **Ve preview del carrito** (botón flotante derecho con total)
5. **Revisa pedido** en `OrderReview` — puede ajustar notas, cantidades, agregar nota general
6. **Confirma** → pedido insertado en Supabase con `estado: 'nuevo'`
7. **Tracking en tiempo real**: `OrderTracking` muestra estado actual con barra de progreso
   - `nuevo` → `preparando` → `listo` → instrucción "Llama al mesonero"
8. Si tiene pedidos activos, el botón verde "Mis pedidos" persiste en el menú

### 5.2 Cocina

1. Accede a `/cocina` (sin login, diseñado para tablet interna)
2. Ve pedidos con `estado: nuevo | preparando` en tiempo real
3. Toca pedido → cambia estado: `nuevo → preparando → listo → entregado`
4. Los cambios se reflejan instantáneamente en el tracking del cliente

### 5.3 Admin

1. Login con email/password en `/login` → sesión Supabase Auth
2. **Menú**: agregar/editar/eliminar platos, subir foto JPG, seleccionar emoji, asignar categoría en dos niveles, marcar disponible/no disponible
3. **Categorías**: crear secciones y subcategorías anidadas, eliminar (elimina subcategorías en cascada)
4. **QR**: genera QR por número de mesa, descargable

## 6. Decisiones Técnicas Relevantes

### Supabase Realtime — canal único por instancia
El hook `useTableOrders` (y el de cocina) appenden `Date.now()` al nombre del canal para evitar el error `"cannot add postgres_changes callbacks after subscribe()"` que aparece cuando React StrictMode monta el componente dos veces en desarrollo.

```ts
const channelName = `table-orders-${restauranteId}-${mesa}-${Date.now()}`
```

### Eliminación optimista en admin
`MenuManager` mantiene un `Set<string>` de IDs eliminados que filtra el array local antes de que Supabase confirme el DELETE. Mejora la percepción de velocidad sin perder consistencia.

### Imágenes en Next.js — hostname exacto
`next.config.ts` usa `remotePatterns` con el hostname exacto de Supabase en lugar de `*.supabase.co` (no soportado por Next.js Image optimization).

### Input de precio controlado
`value={form.precio === 0 ? '' : form.precio}` evita el bug donde escribir "1" con `0` existente resulta en "01".

### API de tasa BCV con ISR
`/api/tasa-bcv` actúa como proxy a `ve.dolarapi.com` con `revalidate: 259200` (72h). Si la API falla devuelve `null` y el cliente omite la conversión a bolívares. No bloquea el flujo del pedido.

### Instalación de dependencias en Vercel
`vercel.json` especifica `"installCommand": "npm install --legacy-peer-deps"` para que emoji-mart instale sin conflictos de peer deps de React 19.

## 10. Consideraciones para Siguiente Fase

### Acciones inmediatas recomendadas
1. **Eliminar Firebase** del proyecto — `npm uninstall firebase` y limpiar `.env.example`
2. **Restringir RLS Storage** — agregar autenticación a la política de INSERT de imágenes (solo admin)
3. **Migrar `categoria` de TEXT a FK** en `menu_items` para mantener integridad referencial

### Escalabilidad
El proyecto usa Supabase free tier con límites de:
- 500 MB de base de datos
- 1 GB de Storage
- 2 GB de ancho de banda
- 200 conexiones concurrentes de Realtime

Para un restaurante pequeño-mediano (< 10 mesas, < 100 pedidos/día) el free tier es suficiente. Para escalar a múltiples restaurantes o volumen mayor, considerar tier Pro (~$25/mes).

### Multi-tenant
El diseño actual soporta múltiples restaurantes (`restaurante_id` en todas las tablas), pero la UI solo expone un restaurante configurable por variable de entorno. La extensión a SaaS multi-restaurante requeriría:
- Selector de restaurante en login admin
- Subdominios por restaurante, o path dinámico `/r/[restauranteId]/mesa/[numero]`
